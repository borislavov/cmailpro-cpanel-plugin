Configurar SpamAssassin

La característica de configuración de SpamAssassin™ le permite colocar en la lista negra (bloquear), y la lista blanca (aprobar), direcciones de correo electrónico, así como también especificar los puntajes que serán utilizados para cada prueba en particular.

¿Qué es una lista negra?

Una lista negra contiene una lista de direcciones, servidores, y contenidos a los que no se le permiten pasar el filtro. Esta opción también envía automáticamente un mensaje de advertencia personalizado al remitente.

¿Qué es un lista blanca?

Una lista blanca contiene una lista de direcciones de correo electrónico, servidores, y contenidos, a los que se le permite el paso a través del filtro.

Agregar direcciones a la lista negra

Ingrese direcciones cuyos correos electrónicos a menudo pasan a través del filtro de correo basura, aunque sin embargo no desea recibir correo electrónico procedente de ellas.

Se puede utilizar * como comodín para múltiples caracteres, o ? como comodín para caracter único.

Ejemplos: usuario@ejemplo.com
*@ejemplo.com
?suario@ejemplo.com

Definir puntaje general de correo basura

SpamAssassin examina todos los mensajes de correo electrónico buscando características de correo basura, y luego les asigna un puntaje general. Ingrese el puntaje requerido para que un mensaje sea considerado correo basura.

5.0 es la definición predeterminada, y es agresiva. Podría ser apropiada para un usuario individual, aunque un ISP debería definir el valor predeterminado de forma más conservadora (8.0 o 10.0).

Definir puntajes de pruebas individuales

Se puede adaptar con precisión al sistema SpamAssassin para su servidor asignando puntajes a pruebas individuales. SpamAssassin utiliza cientos de pruebas, por favor vea su documentación en http://spamassassin.apache.org/tests.html para obtener más información. Necesitará conocer cuál versión de SpamAssassin se está ejecutando su servidor.

Se pueden ingresar puntajes de pruebas individuales en el siguiente formato:
“score” “TEST_NAME” “1 o 4 positivo o números negativos”

Ejemplo: score INVALID_DATE 3.2 3.3 2.5 2.1

Este ejemplo define los puntajes que SpamAssassin asigna a un mensaje con una fecha inválida en su encabezado. Por favor advierta que:

  • Si solamente se menciona un número, ese puntaje siempre es utilizado para la prueba.
  • Definir un puntaje en 0 desactivará la prueba.

En el ejemplo de arriba, 3.2 es el primer puntaje, 3.3 es el segundo, 2.5 es el tercero, y 2.1 es el cuarto. Si se ingresan cuatro números, como en el ejemplo, entonces el puntaje utilizado depende de si Bayes y las pruebas de red están permitidas en su instalación de SpamAssassin:

  • El primer puntaje es utilizado cuando Bayes y las pruebas de red se desactivan.
  • El segundo puntaje es usado cuando Bayes es desactivado, pero las pruebas de red están habilitadas.
  • El tercer puntaje es usado cuando Bayes está activado, pero las pruebas de red están deshabilitadas.
  • El cuarto puntaje es usado cuando Bayes está activado y las pruebas de red están habilitadas.

Agregar direcciones a la lista blanca

Ingrese direcciones cuyos correos electrónicos a menudo son bloqueados, aunque sin embargo usted desea recibirlos.

Se puede utilizar * como comodín para múltiples caracteres, o ? como comodín para caracter único.

Ejemplos: usuario@ejemplo.com
*@ejemplo.com
?suario@ejemplo.com

  • Recuerde hacer clic en Guardar después de ingresar sus configuraciones.
