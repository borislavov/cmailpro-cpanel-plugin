¿Qué son las páginas de error?

Las páginas de error muestran mensajes de advertencia, cuando los visitantes encuentran problemas al tratar de ingresar a su portal de Internet.

Cada tipo de problema posee su propio código de estado HTTP. Los códigos de error HTTP más comunes son:

  • 400 - Solicitud errónea - La solicitud del usuario contiene una sintaxis incorrecta.
  • 401 - Autorización requerida - El archivo solicitado requiere verificación (un nombre de usuario y contraseña).
  • 403 - Prohibido - El servidor no permitirá al visitante ingresar al archivo solicitado. Si un visitante ve este código de error, verifique las configuraciones de permisos de archivo. También se puede verificar si el archivo ha sido protegido mediante el Administrador de índices de cPanel.
  • 404 - Página incorrecta - El servidor no pudo encontrar el archivo solicitado por el visitante. Este error ocurre normalmente cuando un URL es ingresado de forma incorrecta.
  • 500 - Error interno del servidor - Este error significa que el servidor ha encontrado una condición inesperada. Es un error del tipo "capturar todo", que aparece cuando no se puede reunir información específica por parte del servidor en relación a la condición. Este error ocurre con frecuencia cuando una solicitud de aplicación no se puede satisfacer debido a que la aplicación está mal configurada.

Consulte la documentación de cPanel para obtener más información acerca de códigos HTTP.

¿Por qué crear una página de error personalizada?

Crear una página de error personalizada puede ayudar a explicar el error a los visitantes de su portal de Internet. Se puede también incluir diferentes imágenes y tipos de letra, para personalizar sus páginas de error.

Crear una página de error personalizada

  1. Haga clic en el menú Administrar y seleccione el dominio apropiado.
  2. Seleccione un código de error de la pestaña Editar códigos comunes de error, o, para editar una página para un código diferente, haga clic en Mostrar todos los códigos de estado de error HTTP y seleccione un código.
  3. Haga clic en el vínculo del código de error. Aparecerá una nueva página.
