What are MX entries?

MX entries, also known as mail exchanger entries, are resource records in the domain name system (DNS). A domain’s MX records dictate how email is routed through Simple Mail Transfer Protocol (SMTP).

If your mail is delivered to the server that your domain is on, you will not need to edit your MX entries.

However, should you want your mail delivered to an external mail server or another domain, you need to point your MX entry to that location.

This interface allows you to add new MX entries and assign a priority to each MX entry. Priorities dictate which record will be used to route incoming and outgoing email. For example, an MX record with a priority value of ‘0’ will be the first record attempted. If this record fails (due to bad DNS information or an outage), the second MX record (usually with a priority value of ‘5’) will be attempted. This process will continue until messages are delivered to the proper location.

Generally, MX record priority values are divisible by 5, with 0 being the first priority (‘lowest priority’) MX entry.

Note: A valid MX entry is the hostname of a server, formatted like mail.example.com. IP addresses, such as 127.0.0.1, are not valid.

Email routing

This section of the screen allows you to specify what kind of mail exchanger your domain is using. You may select from 4 options:

  • Automatically Detect Configuration — We strongly recommend using this option. Specifying this option allows cPanel to detect what type of mail exchanger your domain should use.
  • Local Mail Exchanger — This option specifies that the mail server associated with your domain will always accept mail. Mail will be delivered locally on the server whether it is sent from your server or another mail server.
  • Backup Mail Exchanger — This option specifies that you wish to use the mail server associated with your domain as a backup mail exchanger. Using this option allows mail to be held at this exchanger until a lower priority exchanger is available.
  • Remote Mail Exchanger — This option specifies that the mail server associated with your domain should not accept mail locally. Instead, mail will be transferred to the lowest-numbered MX record.
  • Make your selection, then click Change.

Add new MX record

1. Enter the priority and destination in the appropriate boxes.

  • Lower values denote higher priority, with 0 being the highest possible priority.
  • The primary mail server(s) (with the lowest-numbered priority) will receive mail sent to your domain.
  • Secondary mail servers (those with higher priority values) can be used for backup or other purposes.
  • If you assign multiple mail servers the same priority, then when that level of mail server is needed, mail will be distributed to those servers randomly.

2. Click Add New Record.

Edit MX record

1. Click Edit to the right of the appropriate record.

2. Enter the priority and destination in the appropriate boxes.

3. Click Edit to save changes or click cancel to cancel changes.

Delete MX record

1. Click Delete to the right of the appropriate record.

2. Click Delete to save changes or click cancel to cancel changes.
