About public key encryption

Encryption is very important for keeping SSH communications private. Public key encryption is a method of communicating using a matching pair of keys (one public, one private) that create a secret code. This ensures that anyone looking at messages sent from or received by your computer will be unable to read them without access to the private key.

A private key is a string of characters that a computer uses to encode or decode encrypted messages. Keeping this key secret is integral to protecting your confidential information.

A public key is a string of characters that a computer uses to encode or decode encrypted messages it receives. Typically, a public key will be placed on a server so you can access that server with encryption.

Generate a new SSH key

  • Click Generate a new Key. You will be directed to a new page.

You will create a challenge password when generating the new key, to add an extra layer of security.

Once you've generated public and private keys, keep your private key safely guarded.

Import an SSH key

If you've already generated a key pair, you can install it using this screen.

  • Click Import Key. You will be directed to a new page.

Delete a public or private key

  • Click Delete to the right of the appropriate key name. You will be directed to a new page.

View or download a public or private key

  • Click View/Download to the right of the appropriate key name. You will be directed to a new page.

Manage public key authorization

  • Click Manage Authorization to the right of the appropriate key name. You will be directed to a new page.
